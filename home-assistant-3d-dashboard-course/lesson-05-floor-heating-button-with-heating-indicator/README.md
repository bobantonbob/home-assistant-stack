# Lesson 05 — Floor Heating Button with Heating Indicator

## Home Assistant 3D Dashboard Course

У цьому занятті ми продовжуємо створювати 3D Dashboard для Home Assistant і додаємо окремий елемент керування для обігріву. Основна задача уроку — зробити кнопку теплої підлоги / обігріву з візуальними індикаціями та адаптувати сам дашборд так, щоб він нормально відображався на екрані.

У результаті заняття ми отримуємо 3D-план квартири, на якому видно кнопку обігріву, температуру, іконку радіатора та окремий червоний індикатор, який показує момент реального нагріву.

## Що ми зробили в занятті 5

У цьому уроці ми створили кнопку обігріву для 3D Dashboard в Home Assistant.

Основна ідея полягала в тому, щоб зробити не просто кнопку `on/off`, а більш інформативний елемент, який показує реальний стан системи.

У Home Assistant сутність типу `climate` може бути увімкнена, але це ще не означає, що вона саме зараз гріє. Термостат може бути активний, але перебувати в очікуванні, якщо поточна температура вже досягла заданої.

Тому ми зробили окрему логіку:

- якщо обігрів вимкнений — іконка сіра;
- якщо термостат увімкнений, але зараз не гріє — іконка помаранчева;
- якщо реально йде нагрів — іконка червона, фон підсвічується, а поряд з’являється червона точка.

---

## Основна кнопка обігріву

Основна кнопка створена через `custom:button-card`.

Вона підключена до сутності:

```yaml
entity: climate.floor_heating
```

Для іконки використовується:

```yaml
icon: mdi:radiator
```

Назва кнопки прихована:

```yaml
show_name: false
```

Стан показується:

```yaml
show_state: true
```

Але замість стандартного стану ми виводимо поточну температуру:

```yaml
state_display: |
  [[[
    const temp = entity.attributes.current_temperature;
    return temp ? temp + '°C' : entity.state;
  ]]]
```

Цей блок бере значення з атрибута `current_temperature` і показує його прямо на кнопці.

Наприклад:

```text
21.5°C
```

Якщо температура з якихось причин недоступна, тоді показується стандартний стан сутності.

---

## Зміна фону кнопки

Фон кнопки змінюється залежно від того, чи відбувається нагрів.

```yaml
background: |
  [[[
    if (entity.attributes.hvac_action === 'heating') {
      return 'rgba(255,165,0,0.65)';
    }

    return 'rgba(0,0,0,0.5)';
  ]]]
```

Якщо `hvac_action` дорівнює `heating`, кнопка отримує помаранчевий фон.

Якщо нагрів неактивний — фон залишається темним і напівпрозорим.

---

## Світіння кнопки

Для візуального ефекту додано `box-shadow`.

Коли система гріє, кнопка має яскраве жовте світіння:

```yaml
box-shadow: |
  [[[
    if (entity.attributes.hvac_action === 'heating') {
      return '0 0 clamp(10px, 2vw, 24px) rgba(255,215,0,0.95)';
    }

    return '0 0 clamp(4px, 1vw, 8px) rgba(0,0,0,0.6)';
  ]]]
```

Це дозволяє швидко побачити на 3D-плані, що обігрів зараз активний.

---

## Колір іконки

Колір іконки також змінюється залежно від стану.

```yaml
icon:
  - width: clamp(18px, 3vw, 30px)
  - color: |
      [[[
        const action = entity.attributes.hvac_action;
        const state = entity.state;

        if (action === 'heating' || action === 'heat' || action === 'running') {
          return '#ff0000';
        }

        if (state !== 'off') {
          return 'orange';
        }

        return 'gray';
      ]]]
```

Логіка така:

```text
червоний   — реально йде нагрів
помаранчевий — термостат увімкнений, але зараз не гріє
сірий      — обігрів вимкнений
```

---

## Червоний індикатор нагріву

Окремо ми створили маленьку червону точку.

Вона також зроблена через `custom:button-card`, але використовується іконка:

```yaml
icon: mdi:circle
```

Цей елемент не є кнопкою керування. Він лише показує індикацію.

Тому натискання вимкнене:

```yaml
tap_action:
  action: none
```

Фон і тінь прибрані:

```yaml
styles:
  card:
    - background: none
    - box-shadow: none
```

---

## Логіка червоної точки

Червона точка з’являється тільки тоді, коли задана температура більша за поточну.

```yaml
color: |
  [[[
    const current = Number(entity.attributes.current_temperature);
    const target = Number(entity.attributes.temperature);

    if (entity.state !== 'off' && target > current) {
      return '#ff0000';
    }

    return 'transparent';
  ]]]
```

Тут ми беремо два значення:

```text
current_temperature — поточна температура
temperature         — задана температура
```

Якщо термостат не вимкнений і задана температура вища за поточну, індикатор стає червоним.

Якщо нагрів не потрібен, індикатор стає прозорим.

---

## Чому ми зробили два окремих елементи

Кнопка обігріву складається з двох частин:

1. Основна кнопка з температурою та іконкою.
2. Маленька червона точка-індикатор.

Це зручніше, ніж намагатися все зробити в одному елементі, тому що червону точку можна окремо позиціонувати на плані.

Основна кнопка стоїть тут:

```yaml
style:
  top: 86%
  left: 57%
```

Червона точка стоїть трохи вище і правіше:

```yaml
style:
  top: 84%
  left: 59%
```

Завдяки цьому індикатор виглядає як частина кнопки, але ним легше керувати окремо.

## Адаптація 3D Dashboard під екран

Після створення кнопки ми почали адаптувати сам дашборд.

Проблема була в тому, що звичайна картка `picture-elements` у Home Assistant може виглядати занадто маленькою або навпаки розтягуватися так, що з’являється горизонтальний або вертикальний скрол.

Також квадратний 3D-план не завжди правильно розтягується на широкий екран. Якщо просто розтягнути його по ширині, він може вилізти вниз і обрізатися. Якщо збільшити тільки висоту секції, з’являється пусте місце.

Тому ми перейшли до правильнішого підходу.

## Створення нової вкладки

Для цього заняття ми створили окрему вкладку в Home Assistant.

Ідея така: краще винести 3D Dashboard в окрему вкладку, щоб не змішувати його з іншими картками.

Під час створення вкладки ми обрали макет:

```text
Horizontal (layout-card)
```

Тобто горизонтальний макет.

Такий підхід дозволяє розмістити:

```text
ліва частина  — 3D-план квартири
права частина — додаткова панель керування або інформаційні блоки
```

У нашому випадку основний акцент залишається на лівій частині, де знаходиться 3D-план.

---

## Чому не просто Panel View

Ми пробували варіант з повноекранною панеллю, але для квадратного 3D-плану це не завжди зручно.

Якщо змусити картинку займати всю ширину екрана, вона може не влізти по висоті. Через це з’являється обрізання або скрол.

Тому більш стабільний варіант — використовувати `layout-card` і задати окрему ліву область під 3D-план.

---

## Основна логіка layout-card

У повному файлі дашборду використовується:

```yaml
type: custom:layout-card
layout_type: custom:grid-layout
```

Далі задається сітка:

```yaml
layout:
  grid-template-columns: minmax(0, min(calc(100vw - 460px), calc(100vh - 120px))) minmax(320px, 420px)
  grid-template-areas: |
    "left right"
  gap: 12px
  justify-content: center
  align-items: start
```

Тут головна частина:

```yaml
min(calc(100vw - 460px), calc(100vh - 120px))
```

Це означає, що розмір лівої частини буде рахуватися не тільки від ширини екрана, а й від висоти.

Завдяки цьому квадратний 3D-план не обрізається і не створює зайвий скрол.

---

## Ліва частина — 3D-план

3D-план розміщується в області `left`:

```yaml
view_layout:
  grid-area: left
```

Картка має тип:

```yaml
type: picture-elements
```

І використовує зображення:

```yaml
image: /local/home11.png
```

Також важливо вказати квадратне співвідношення сторін:

```yaml
aspect_ratio: 1/1
```

Це потрібно тому, що наш 3D-план має квадратну форму.

---

## Права частина — додаткова панель

Права частина може використовуватися для додаткових елементів.

Наприклад:

- сенсори;
- графіки;
- кнопки;
- інформаційні картки;
- стан інвертора;
- керування освітленням;
- показники споживання.

У цьому занятті головна увага була на лівій частині та кнопці обігріву, але структура вже дозволяє розвивати дашборд далі.

---

## Адаптація для мобільного екрана

У layout-card також додано адаптацію для малих екранів:

```yaml
mediaquery:
  "(max-width: 900px)":
    grid-template-columns: 1fr
    grid-template-areas: |
      "left"
      "right"
```

Це означає, що на вузькому екрані ліва і права частина стануть не поруч, а одна під одною.

На комп’ютері:

```text
left | right
```

На телефоні:

```text
left
right
```

## Позиціонування елементів

Усі елементи на 3D-плані позиціонуються через відсотки:

```yaml
style:
  top: 86%
  left: 57%
```

`top` відповідає за положення зверху вниз.

`left` відповідає за положення зліва направо.

Якщо кнопка стоїть не там, де потрібно, змініть ці значення.

Наприклад, щоб перемістити кнопку трохи вище:

```yaml
top: 84%
```

Щоб перемістити правіше:

```yaml
left: 60%
```
