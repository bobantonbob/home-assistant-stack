## Куди завантажувати зображення в Home Assistant

Щоб Home Assistant міг бачити це зображення, його потрібно завантажити в папку:

```text
/config/www/
```

Наприклад:

```text
/config/www/home11.png
```

Після цього в Home Assistant це зображення буде доступне через шлях:

```text
/local/home11.png
```

Важливо: папка `/config/www/` у Home Assistant відповідає публічному шляху `/local/`.

Тобто якщо файл лежить тут:

```text
/config/www/home11.png
```

то в dashboard-коді ми вказуємо так:

```yaml
image: /local/home11.png
```

## Базовий приклад картки picture-elements

Нижче приклад мінімальної картки, яка показує 3D-зображення квартири як фон:

```yaml
type: picture-elements
image: /local/home11.png
aspect_ratio: 1/1
elements: []
```

Це ще не повноцінний дашборд, а тільки основа. Далі поверх цього зображення ми будемо додавати кнопки, датчики, розетки, світло, камери, інвертор, енергоспоживання та інші елементи.

## Що таке picture-elements

`picture-elements` — це картка Home Assistant, яка дозволяє розміщувати елементи поверх зображення.

Наприклад, поверх плану квартири можна додати:

- кнопку світла;
- датчик температури;
- розетку;
- сенсор потужності;
- камеру;
- статус охорони;
- заряд батареї;
- стан інвертора;
- будь-які інші сутності Home Assistant.

## Примітка

Назву файлу краще зробити простою:

```text

```

Так буде зручніше використовувати його в YAML:

```yaml

```

Якщо файл називається `home11 (2).png`, тоді у шляху можуть з’явитися пробіли та спеціальні символи. Для Home Assistant і GitHub краще використовувати короткі назви без пробілів, дужок і кирилиці.
