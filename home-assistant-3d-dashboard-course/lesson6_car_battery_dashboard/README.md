# Заняття 6 | Додаємо індикацію батареї авто на 3D Dashboard Home Assistant

У цьому занятті додаємо на 3D Dashboard в Home Assistant індикацію батареї електромобіля та кнопку заряджання.

Ідея проста: на картинку дашборду накладаємо живі елементи Home Assistant через `picture-elements` і `custom:button-card`.

## Що робимо сьогодні

1. Додаємо кнопку заряджання авто.
2. Виводимо поточну потужність зарядки у `kW`.
3. Додаємо окремий індикатор батареї авто у `%`.
4. Робимо напівпрозорий фон, підсвітку, рамку та адаптивний розмір.
5. Розміщуємо все поверх картинки авто через `top` і `left`.

## Файли YAML

### `yaml/01_charging_button.yaml`

Це кнопка заряджання авто.

Вона використовує:

```yaml
entity: switch.tesla_switch
```

І показує потужність заряджання з сенсора:

```yaml
sensor.anma_charger_power_2
```

Логіка:

- якщо сенсор потужності доступний — показує значення у `kW`;
- якщо сенсор недоступний — показує `ON` або `OFF`;
- якщо зарядка увімкнена — кнопка світиться зеленим;
- якщо зарядка вимкнена — кнопка темна.

### `yaml/02_battery_percent.yaml`

Це індикатор заряду батареї авто.

Він використовує:

```yaml
entity: sensor.anma_battery_level
```

Логіка:

- якщо сенсор доступний — показує заряд у відсотках;
- значення округлюється до цілого числа;
- якщо сенсор недоступний — показує `--%`.

### `yaml/03_picture_elements_example.yaml`

Це приклад, як обидва блоки можуть виглядати всередині `picture-elements`.

Його можна використовувати як стартовий шаблон, але перед запуском треба перевірити шлях до картинки та свої назви сутностей.

## Необхідні компоненти Home Assistant

Потрібно мати встановлений:

- `custom:button-card`
- картку `picture-elements`
- сенсор заряду батареї авто
- сенсор потужності заряджання
- switch або іншу сутність для керування зарядкою

## Що треба змінити під себе

Якщо у вас інші назви сутностей, замініть їх у YAML.

Наприклад:

```yaml
switch.tesla_switch
sensor.anma_charger_power_2
sensor.anma_battery_level
```

на свої:

```yaml
switch.your_charger_switch
sensor.your_charger_power
sensor.your_car_battery_level
```

## Куди завантажувати картинки

У Home Assistant картинки зазвичай кладемо у папку:

```text
/config/www/
```

Наприклад:

```text
/config/www/dashboard/car_example.png
```

Тоді у Lovelace шлях буде такий:

```yaml
image: /local/dashboard/car_example.png
```

## Як рухати елементи на дашборді

Позиція задається через:

```yaml
style:
  top: 89%
  left: 20%
```

- `top` — відступ зверху;
- `left` — відступ зліва.

Якщо індикатор стоїть не там, де потрібно, міняємо ці два значення.

## Чому використовується `clamp()`

`clamp()` дає адаптивний розмір.

Наприклад:

```yaml
width: clamp(52px, 7vw, 78px)
```

Це означає:

- мінімальна ширина — `52px`;
- бажаний розмір — `7vw`;
- максимальна ширина — `78px`.

Так елемент краще виглядає і на великому моніторі, і на телефоні.

## Текст для пояснення у відео

Спочатку додаємо кнопку заряджання авто. У моєму випадку це `switch.tesla_switch`. При натисканні кнопка подає або знімає заряд. На цій же кнопці ми одразу показуємо поточну потужність заряджання у кіловатах із сенсора `sensor.anma_charger_power_2`.

Після цього додаємо окремий індикатор батареї авто. Це вже не кнопка керування, а інформаційний блок. Він бере дані із сенсора `sensor.anma_battery_level`, округлює значення і показує заряд у відсотках прямо поверх картинки автомобіля.

Далі робимо стиль: напівпрозорий чорний фон, бірюзову рамку, легке світіння і адаптивний розмір через `clamp()`. Завдяки цьому дашборд виглядає не як суха таблиця, а як нормальний живий 3D-інтерфейс.
